## 🎯 改善提案一覧（優先度順）

### P0: 収益化（最優先）

#### 1. 会計ソフト・税務サービスのアフィリエイト化
**目的**: 計算結果後の「実行ステップ」として会計ソフト利用を促進→アフィリエイト報酬獲得

**仕様**:
- 計算結果の下部に「📝 次のステップ」セクションを追加
- 計算結果の金額に応じて以下を提案:
  - 確定申告が必要な場合: freee/マネーフォワード/弥生会計へのA8アフィリエイトリンク
  - iDeCo/ふるさと納税が多い場合: SBIネオモバイル証券などへのアフィリエイト
- 現在の内容に「🔗 節税に役立つツール」があるが、ここを強化して実装
- A8で各サービスのアフィリエイトが利用可能か確認後、追加

**実装タスク**:
- HTML結果セクションに「calcResult`下部に「次のステップ・おすすめツール」セクション追加
- 計算ロジックで判定して動的に推奨ツールを表示（白/青色申告、iDeCo金額、ふるさと納税額による分岐）
- A8のアフィリエイトリンク（freee, マネーフォワード, 弥生会計など）を統合
- GA/GTMで「おすすめツール」クリック数を追跡可能に

**DoD**:
- [ ] 「次のステップ」セクションがモバイル・PCで適切に表示される
- [ ] 最低3つのアフィリエイトサービス（freee, マネーフォワード, SBIなど）が動的に表示される
- [ ] A8報酬が発生する状態で公開される
- [ ] 既存A8リンク（求人、ハピタス）と統一された見た目

---

#### 2. Amazon/A8アフィリエイトリンクの配置最適化
**目的**: 現在の「節税Tips」内のアフィリエイトリンク→結果表示時にも高視認性で提示

**仕様**:
- 現在: Tips タブ内にのみ Amazon/A8 リンク配置
- 改善: 計算結果ヒーロー下部に「💡 あなたにおすすめの書籍・サービス」バナーを追加
  - Amazon 書籍: 青色申告の本、副業節税本
  - A8 サービス: 転職/バイト/マネー系
- 計算結果（確定申告要/不要、税率など）に応じてフィルタリング

**実装タスク**:
- 計算結果エリア（#calc-result内）に新セクション「おすすめリソース」を追加
- 計算ロジック連動: 青色申告選択時→青色申告の本をおすすめ、など

**DoD**:
- [ ] 計算結果表示時、アフィリエイトリンクが自動提示される
- [ ] モバイルで最低1列、PC で2列以上の美しいレイアウト
- [ ] クリック追跡が GTM で可能

#### 4. 内部リンク戦略の強化
**目的**: 関連ページへの流動→ドメイン評価向上・ユーザー滞在時間増加

**仕様**:
- 現在: 「関連サービス」セクション（現在は「Tips」タブと「ガイド」タブ）に限定
- 改善:

**実装タスク**:

- 副業タイプ（ブログ、せどり、ハンドメイド等）に応じて異なるリンク提示
- 既存ブログ記事（/blog/sidejob-tax-guide など）への直接リンク

**DoD**:
- [ ] 計算結果表示時に、最低3つの内部・外部リンクが動的に表示される
- [ ] GA で「関連リンク」クリックが追跡可能

### P2: ユーザー継続

#### 7. 「次のステップ・アクションプラン」の追加
**目的**: 計算結果をもとに、ユーザーが実際に取るべき行動を提示→継続利用・行動喚起

**仕様**:
- 計算結果表示後に、「📋 あなたの申告・節税チェックリスト」セクション追加
- ロジック:
  - 確定申告が「必要」→「① 2月15日までに確定申告」「② 源泉徴収票を準備」等の TODO リスト
  - 確定申告が「不要」→「① 住民税申告（市区町村へ）」「② 普通徴収を選択」等
  - iDeCo 金額が大きい→「① SBI か楽天でiDeCo口座開設」のステップ
  - ふるさと納税を使用→「① 控除上限内でふるさと納税を実施」
- 各ステップにはリンク（freee のログイン画面、e-tax ページ等）を統合

**実装タスク**:
- 計算ロジック内で判定ロジックを追加
- HTML 結果セクション内に「TODO・チェックリスト」を動的生成
- CSS で進捗バーか checkBox スタイルで視覚化

**DoD**:
- [ ] 計算結果表示時に、ユーザーの条件に応じた 3〜5 個のアクションが提示される
- [ ] チェックボックスは localStorage に保存可能（※オプション）
- [ ] モバイル・PC で見やすくレイアウトされている

---

#### 8. 計算履歴・比較機能の追加（オプション）
**目的**: 複数シナリオの比較→サービスへの粘着性向上

**仕様**:
- 「複数シナリオを比較したい」ユースケース対応
- 計算結果下に「💾 この結果を保存」ボタン追加
- 保存した結果は、新しい計算時に「📊 比較表示」で並べられる
- 実装: localStorage で複数結果を配列で保存、比較タブで表示

**実装タスク**:
- 計算結果に「保存」ボタン追加（クリックで現在の入力値+結果を localStorage に保存）
- 比較タブを新設（「保存結果」）し、複数結果を横並びで表示
- 削除機能も搭載

**DoD**:
- [ ] 複数の計算結果が保存・比較できる
- [ ] localStorage 容量内に収まっている（5MB 以下）
- [ ] ページリロード後も保存が維持される
