## 🎯 残りの改善提案（優先度順・60分内で実装可能）

### P1: SEO・集客・内部リンク強化

#### 1. 内部リンク戦略の最適化 - 関連ツール・計算機への誘導
**目的**: ドメイン内の回遊を増やす → 滞在時間増加 → SEO向上 + 広告インプレッション増加

**現状**:
- 「関連情報」セクションに内部ブログリンク 2-3個
- 関連ツール（BMI計算機、家計簿診断など）への直接リンクなし
- 同一ユーザーが複数ツールを使う確率が低い

**改善案**:
- 計算結果下に「あなたにおすすめの関連ツール」セクション追加
- 副業タイプ・年収帯に応じた関連ツール動的推奨
  - フリーランス・個人事業主 → 「事業所得計算機」「経費管理ツール」
  - アルバイト・給与 → 「手取り計算機」「給与明細チェック」
  - 株・投資 → 「FIRE計算機」「資産運用診断」
  - 不動産・出費削減 → 「家計簿診断」「ローン返済シミュレーター」
- 各ツールのサムネイル画像（50x50px）+ 説明文（1行）+ リンク
- トラッキング: GA で「related_tool_click」イベント記録

**実装タスク**:
- HTML: `<section>あなたにおすすめの関連ツール</section>` を結果セクション直下に追加
- JavaScript: calcResult内で副業タイプ・年収帯判定 → ツールリスト動的生成
- CSS: グリッド表示（3カラム、モバイル時1カラム）

**DoD**:
- [ ] 副業タイプごとに異なるツールが表示される
- [ ] モバイル・PC で見やすくレイアウトされている
- [ ] GTM で「related_tool_click」が追跡される

#### 2. meta description・OGP タグの最適化
**目的**: 検索結果・SNSシェア時のクリック率向上

**現状**:
- meta description: 「副業の手取り額・所得税・住民税を無料でシミュレーション。確定申告が必要かどうかも即判定。11種類の副業に対応した無料計算機。登録不要。」
  → 長い、キーワード重複感あり
- og:description: 同上の別表現
- Twitter Card: あるが、内容が薄い

**改善案**:
- meta description: 「副業の手取り額・税金・確定申告判定を無料計算。月収5万〜100万円対応・20万円ルール判定・青色申告比較。ブログ・せどり・アルバイト等11種類対応。」
  → キーワード追加: 月収幅、ルール判定、複数副業タイプ
- og:description: 「副業収入から税負担を即計算。手取り額・所得税・住民税・確定申告必要性が分かります。フリーランス・アルバイト・投資など全副業対応。完全無料。」
  → より簡潔で claim-focused
- Twitter description: 「副業でいくら税金がかかる？所得税・住民税・確定申告の必要性を無料計算。」
  → 一文で要約

**実装タスク**:
- HTML: `<meta name="description">` / `<meta property="og:description">` / `<meta name="twitter:description">` 更新

**DoD**:
- [ ] description が 155 字以内（Google推奨）
- [ ] og:description が 60-120 字（SNS最適）
- [ ] 主要キーワード（手取り・税金・計算・無料・20万円）を含む

#### 3. breadcrumbList スキーマの改善
**目的**: 検索結果での「階層表示」化 → CTR向上

**現状**:
- BreadcrumbList スキーマあり（ホーム > カテゴリ > このページ）
- だが、「副業の種類ごとの詳細ページ」がないため、階層が浅い

**改善案**:
- breadcrumbList に「副業税金 FAQ」「シミュレーション方法」などのサブページを示唆
  - ホーム > 計算機 > 副業・税金 > **このページ**
- または、カテゴリを「税金」「副業」「計算機」に分割表示

**実装タスク**:
- JSON-LD breadcrumbList の itemListElement を 3項目 → 4項目に拡張

**DoD**:
- [ ] breadcrumbList が 4階層以上（SEO評価向上）

### P2: ユーザー継続・再訪導線

#### 4. 「計算結果をメール送信」ボタン追加
**目的**: ユーザーの再訪・ブックマーク化 → リピート利用

**現状**:
- 「コピーボタン」「シェアボタン」あり
- だが、ユーザーが結果を「自分のメール」に保存する方法がない

**改善案**:
- 計算結果の上部に「📧 この結果をメール送信」ボタン追加
- クリックで `mailto:` リンク開く（ユーザーのデフォルトメールクライアント）
- メール本文テンプレート:
```
件名: 副業税金シミュレーター結果
本文:
【あなたの計算結果】
副業種類: [副業タイプ]
月収入: [月収]
経費率: [経費率]
手取り額: [手取り]
確定申告: [必要/不要]

詳細: https://appadaycreator.com/sidejob-tax-simulator/
```

**実装タスク**:
- HTML: 結果セクションに「メール送信」ボタン追加
- JavaScript: メール本文 URL エンコード + `window.location.href = 'mailto:...'`

**DoD**:
- [ ] メール送信ボタンが表示される
- [ ] クリックでメールクライアント開く
- [ ] 計算結果が本文に含まれる

### P3: コード品質・アクセシビリティ

#### 6. 画像の alt 属性補充・SVG icon のアクセシビリティ
**目的**: スクリーンリーダー対応 + WCAG 2.1 AA 準拠

**現状**:
- 絵文字（💰、📋、✅）使用 → alt 属性不要
- ただし、アイコン画像があれば alt 属性が必要

**改善案**:
- 既存の img タグすべてに alt 属性を確認・追加
- SVG icon に role="img" + aria-label 追加

**実装タスク**:
- HTML: `<img>` に alt 属性
- SVG: `role="img" aria-label="説明"`

**DoD**:
- [ ] axe DevTools で major issue なし
- [ ] WAVE でアクセシビリティエラーなし
